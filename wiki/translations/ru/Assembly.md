# Assembly/ru
## Введение


{{TOCright}}

Во FreeCAD слово \"[Сборка](Assembly/ru.md)\" обычно используется для обозначения [3D модели](model/ru.md), состоящей из нескольких различимых частей, которые в некотором роде складываются вместе для создания функционального объекта, точно так же, как создаются реальные изделия.

Например, болт, шайба и гайка - это три отдельных тела, которые при соединении составляют сборку.

<img alt="" src=images/PartDesign_Body_contiguous_separate.png  style="width:" height="200px;"> <img alt="" src=images/PartDesign_Body_contiguous_assembly.png  style="width:" height="200px;">



*Слева: три отдельных смежных тела, каждое из которых моделируется как [тело PartDesign](PartDesign_Body/ru.md). Справа: три отдельных тела, собранных вместе внутри [Std Part](Std_Part/ru.md) для создания сборки.*



## Применение



### Сборка вручную 

в общих словах, Вам не нужны специальные инструменты для создания сборок, Вам достаточно иметь различные [тела](Body/ru.md), как-то организованные.

Чтобы расположить тела там, где они вам нужны, вы можете

-   использовать инструмент [Std Преобразование](Std_TransformManip/ru.md),
-   использовать диалоговое окно <img alt="" src=images/Std_Placement.svg  style="width:16px;"> [Std Расположение](Std_Placement/ru.md), или
-   изменить параметр [placement](Placement/ru.md) непосредственно в [редакторе свойств](Property_editor/ru.md).

Вы можете использовать один из псевдо-сборочных [внешних верстаков](external_workbenches/ru.md), вроде Lattice2, Manipulator, Part-o-magic или WorkFeature, чтобы помочь себе найти пересечения, померить расстояния и распределить ваши объекты нужным образом.

В целом, объект **[<img src=images/Std_Part.svg style="width:16px"> [Std Part](Std_Part/ru.md)** был разработан как базовый строительный блок для получения сборок. Этот объект используется для группировки нескольких [тел](body/ru.md) и их перемещения как единицы измерения, то есть как подсборки. Затем эту подсборку можно разместить рядом с другими подсборками или использовать в них для создания окончательной сборки.



### Сборка с ограничениями 

Вы можете так же использовать специализированный верстак для сборок, вроде <img alt="" src=images/A2p_workbench.svg  style="width:24px;"> [A2plus](A2plus_Workbench/ru.md), <img alt="" src=images/Assembly3_workbench_icon.svg  style="width:24px;"> [Assembly3](Assembly3_Workbench/ru.md) или <img alt="" src=images/Assembly4_workbench_icon.svg  style="width:24px;"> [Assembly4](Assembly4_Workbench/ru.md). Пожалуйста, учтите, что [Assembly2](Assembly2_Workbench/ru.md) больше не поддерживается, и не рекомендован для новых моделей.

Верстаки для сборок используют ограничения и выражения для создания отношений между объектами в Вашей модели, чтобы математически привязать объекты к месту, например, \"эта плоскость должна прилипнуть к другой плоскости\", \"этот цилиндр должен быть по одной оси с этой окружностью\", \"эта точка должна быть на этой кромке\" и так далее.

Это продвинутое использование программного обеспечения, которое обычно используется в сложных механических системах. Если Ваша [модель](model/ru.md) не слишком сложна, то использование верстаков для сборок может не потребоваться.



## Примечания

Что касается FreeCAD 0.19, в нём нет официального верстака сборок, включённого в систему по умолчанию. Верстаки сборок слишком сложны для программирования из-за многочисленных проблем, которые надо решить относительно эффективного использования [тел](Body/ru.md) и [деталей](Part/ru.md) в Вашей модели. Тем не менее, ситуацию улучшило введение объекта [App Link](App_Link/ru.md).

Обратите внимание, что рабочие столы для сборки обычно несовместимы друг с другом. Если Вы создадите сборку с одним из них, Вы должны будете остаться на нём, и не сможете использовать другой верстак сборок для работы с тем же документом.

Разработка верстаков сборок продолжается, и ожидается, что в какой-то момент какой-либо из них будет введён как \"официальный\". Это может произойти путем продвижения одного из существующих сборочных стендов или их объединения для получения более полного решения.
